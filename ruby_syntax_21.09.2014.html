I have been programming in ruby for few months now. Initially I just coded some Project Euler problems but now I am using ruby quite often for the Tealeaf Academy projects.

I am starting to get confortable with the language but I must admit that I had a few surprises at the beginning. Here I am sharing some:

#1. Where are my functions??.. Ah.Methods ... what?

In Ruby "methods" are used to encapsulate tasks or behaviors, much like functions in other programming languages (Python, Matlab). They are also defined in a similar way.

def method_name() 
  "Some Code"
end

So it is probably just a naming convention that Rubyist adopted for some reason, other than that I do not see any relevant difference in their FUNCTIONality compare to functions :).

#2. What's wrong with the good old fashion for statement?

In order to go through all the elements of an array, hash or string Rubyist usually use the “each” method. As shown below:

Array.each do |element| 
  if element == "pretty"
    puts element
  end
end

This is somewhat convenient because it saves you the time of having to expresively calculate the length of the array, as I have done in numerous ocasions when using other languages.

But what if we just want to repeat an operation a certain number of times. We can still use the each method in combination with ruby ranges e,g (1..3), no problem. But if we want to count backwards instead, flipping the range (3..1) won't work! One has to use the downto()method to get the work done.

Counting Forward Counting Backwards 
(1..3).each do |number| 
  p number
end

=> 1 2 3 

3.downto(1) do |number| 
  p number
end

=> 3 2 1

Ohh well, I guess counting downto(1) makes more “sense” semantically than “for-ing from 3 to 1”, but it’s a bit counterintuitive if one has some previous programming experience.

#3. What's wrong with the good old fashion while statement?

From what I have seen so far the literal while loop is still more used in ruby than its for counterpart, nevertheless it has many other variations that I had not seen before.

One of my favorites is the begin loop that allows you to iterate through the loop at least one time before checking if a certain condition is met.

begin 
  "some code that will run until condition is met"
end until condition
