shoulda-matchers are very helpful "one-liners that test common Rails functionality" such as validations and model associations.

In a previous post I showed how we could test if a todo model was validating its attributes by asserting that the count of todos was zero after attempting to save (for the case when some of the attributes validated were missing).

As expected shoulda-matchers provide an easier way to test this. In order to implement shoulda-matchers we need to:

1. Install the gem

#gemfile
group :test do
  gem 'shoulda-matchers', require: false
end

2. Write your tests!

If for instance, we have a model video that requires the presence of a :title and a :description and also have_many :comments and belongs_to :user we can just simply write the following:

#spec/models/video_spec.rb
it {should belong_to(:category)}
it {should have_many(:comments)}
it {should validate_presence_of(:title)}
it {should validate_presence_of(:description)}

This is a huuuge!! space and time saver.
