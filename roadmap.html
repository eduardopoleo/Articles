I finally put together my blog app which consists of a modified Bootstrap template on top of a Ruby on Rails backend structure. This is not too bad considering that I did not know what a HTML file was 4 months ago. Looking back I realize how simple the application is. It could probably get done in a couple of hours by a half-decent developer. But well, one must go through a learning curve before becoming proficient at anything. I want to list the resources that helped me grasp a better understanding on how the web works.
  1. Lear to Code from scratch. I believe this roadmap is a good source of links and material for anyone looking to get started on web development. It also lays out a reasonable path to gradually build up your skills without getting overwhelmed.
  2. CodeAcademy. HTML&CSS. I am not a huge fan of code academy, although I think they are a valid starting point if you have no idea whatsoever on the topic at hand.
  3. Foundations of Html. This is a nice article put together by the people of Udemy that provides a higher view understanding of the most relevant html5 tags.
  4. 30 Days to Learn HTML & CSS. This tutorial is priceless and still a free resource! Not only guide you through the most important aspects of html/css but also gives you a whole bunch of practical advices, like which text editors and browsers to use, how to use the built-in browserâ€™s Dev tools, which plugins to install, introduces you to css frameworks and psd slicing, it is just great!
  5. Introduction to Computer Science and Programming Using Python. Any decent coding course will do, it will probably do best to take a course in the language you are most interested in.
  6. Ruby on Rails Tutorial and Getting started with Rails guide. Here I would recommend to go over the first chapter of the rails tutorial that teaches you how to set everything up and then jump straight to the getting started guide without keep working on the book.
  7. Have fun when learning.
I must note that I did not discover (neither use) these resources in the order presented above, although if I could go back this is the order I would have chosen. I actually started out with the Edx course and then realized that WebDev was a great and accessible way to put coding to use. Essentially the idea is to understand html and css first and then use an integrated framework (such as rails) to help you craft apps and put everything together. On the way you will realize that you need to code a bit ( simple loops, conditionals, etc) so that is why I included the edx course, but this last could be easily replaced by a codeacademy course on the language you are interested in (e.g python if you are interested in learning Django etc). So what am I still missing?. The obvious answer is Javascript. I planning to hone my javascript skills with this course. I hope to integrate this with what I already know by creating an interactive to do list and a personal website for a friend. Thanks for reading!
