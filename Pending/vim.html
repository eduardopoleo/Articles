Vim tutor

Like everyone else I started my vim journey using vimtutor. Vimtutor introduces you to the most fundamental vim concepts and commands. I could not remember much after going through the tutor for the first time (there is just so much content) but it gave me a good feel of for the editor.

I came back to the tutor once I have been using vim for a while I realized that it actually teaches you about 95% of the editing commands you'll be using in your day by day development. Among some of the concepts the tutor goes trough are:

  • Vim's different modes
  • Many different editing task (appending, inserting, changing, copy/pasting, etc)/li>

Plugins

Vim is a really powerful tool on it's own but plugins make your life much easier specially if you are doing something specific like rails.

How to install them?

Like everything in Vim there are several ways one can do this although repos owners tent to recomend using any of these two plugin managers: Pathogen or Vundle. I settled for Pathogen for no particular reason (probably because it was the first one I came accross with), and it has worked great so far so I have not had the need to switch, but I suggest you to also take a look Vundle because as I said it is also very popular.

If you decide to use Pathogen all you have to do is to clone the plugin's repo into your ~/.vim/bundle directory and you will have it installed. Conversively, if you ever want to remove a specific plugin you just have to remove the corresponding plugin folder, and that's it it's gone!.

File navigations pluginsheader

As I intended to use Vim mostly for rails development, before actually doing editing I most concern with the directory navegation. Becuase after all, how can you confortably edit a file you do not have access to. Here is a list of plugins I usually use for file navegation.

NERDtree

This plugins triggers when typing :NERDtree and it displays a project navegation panel that shows up at the left side of the editor window (much like Sublime's files directory). The plugin allows you to navigate through the current project directory, add, remove and rename documents and also open files on a split.

CtrP

This plug-in triggers by pressing Ctr+P in normal mode you then type the files name and the return key and the file will be open in your current working panel. The plugin also give you some suggestions as you type. By default the file you are opening will replace the file you are currently working on but there are some commands that allow you to open new files on vertical or horizontal splits.

Vim rails

Vim rails is a very powerful plugin that among other things offers easy navegation through the rails directory. You can easily bring up a file just by typing :Rfile_type key_word. For instance, if you want to work on your user model or controller all you have to do is to type :Rmodel user or :Rcontroler users and hit the return key. This also applies to spec, migrations, mailers, layouts, views actually everything.

Vim rails also allows you to quickly generate files and then run them from withing the editor itself! which is pretty cool. For instance a commom workflow for me is to generate a migration by: :Rgenerate migration create_users then :Rmigrations create_users to bring it to my current panel, write a bit of code and finally :Rake. The :Rake command will run the migration from within the editor itself!

You can also use :Raketo run test, see your routes etc. Vim rails offers a lot of other functionalities but these are the ones I use most frequently.

.vimrc

After installing these plugins I had pretty much solved my file navigation needs I could start focussing on improving my file editing workflow. Nevertheless, it was really annoying that some triggers I was using very frequently (e.g NERDtree) were really long and difficult to type, that's when I discovered the .vimrc file.

The .vimrc file is vims configuration file and it usullay lives on your home directory. There you can basically set some defaults for your day to day vim usage, set abreviation, map key and comand combinations to your liking etc. I solved the specific problem of the NERDtree by ca tr NERDTree this basically sets an abreviation so that writting tr is equivalent to typing the full NERDtree command (only when I am on command mode).

Other things that I have set up on the .vimrc include: automatic indentation to 2 spaces, quick copy/pasting to the and from the clipboard automatic bracket, parenthesis... closing among other defaults.

When I just had discovered the .vimrc I imidiatelly started overriding lots of the default commands that I thought were not very useful to me. But as I'm reading and learning more about vim I've been returnig back to the "factory settings". Nonetheless, if you are reading this and you are still planning to get started with vim I highly encourage you to experiment like I did. This will allow you to better understand why things were initially put that way.

Tmux

By this point when I open my computer my routine was this: open a terminal, naviage to my root directory and open the editor, repeat this three more times for three more times for the console, database and server.

Moreover everytime I wanted to switch between some of this windows I had to either click it with the mouse on the lower bar (lame) or toggle using the Alt-tap which is the method i prefer. The problem with this is that is not really obvious which tab is the want you are looking for and many times i found me self going to the full panel of options a few times until if got the window I wanted.

Tmux help you manage this issue by grouping all your terminal windows within a session (which is dispaly as a single window). From within this session you can easily swap to different terminals by pressing some convinietn triggers commands. Also Tmux layout different terminal outputs in the same window. Tmux as vim is also very customizable and also has it own configuration file. Also and probably most importantly you can automate all this configurations using bash scritps, On my next article I am going to show how to do this.
